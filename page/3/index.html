摆脱哑巴英语笔记

发表于 2016-11-01   |   分类于 笔记   |  

笑来老师关于哑巴英语的笔记，源自人人都能学英语

肚子里没货

  • 口语差不在于你不会说，而在于你没什么话可说。就是没有积累，没有输入便无法输出。

市面上的口语书不适合自己

  • 很多口语书挑选了所有的人（共同）最常用的句子，那些句子实际上是人们最少说，甚至根本不说的句子。书籍是对大众写的，不是对个人写的。口语书认为，因为它们是最经常重复出现的，所以它们是最重要的。这就是犯了逻辑错误，其实废话和寒暄的话总是经常出现的，而这些根本不重要。

考虑太多，积累太少

  • 最开始无法张嘴说英语，往往不是因为自己笨，而是因为自己太聪明。笨的人表达需求简单，见人只想（也只能）讨论简单的问题，见到陌生人问“What’s your name?”一点都不觉得自己说的有什么不妥。而聪明的人往往思考很多，无法表达。他们思维复杂，而复杂的思维需要借助复杂的表达才能得以传递。因为表达需求太过复杂，所以导致同样的训练在他们身上根本无法发挥同样的作用──他们事实上需要更多更强的训练。另外，他们很难遇到有能力与他们探讨有深度的问题的人，所以，他们很少有机会去在实践中锻炼自己的口语能力──相对来看，那些头脑简单的人就幸福得多：随便抓来一个人就可以开始练了。
  • 但是口语流利的最终是聪明的人，因为说话本身是一种高度协调的系统工程：要持续输入（捕获对方一切反馈），要高速处理（所谓见机行事可不是那么容易的事情），还要有效输出（说准说对说得精彩更不容易）……这一切，都要有长期的锻炼和积累才行。

微信小程序展望

发表于 2016-11-01   |   分类于 Wechat   |  

听了知乎live中阿婵分享的微信小程序展望。有以下笔记：
微信为什么要做小程序？
1、微信已经是一个超级入口，商业的贪婪性逐利性这些都让微信有作为os的可能。
2、微信通过订阅号，服务号，企业号和小程序希望占据用户更多时间，占据更多应用场景，占据更多服务入口。
为什么要放弃传统app开发？
1、app生态已经相对饱和，红利期已经过去。
2、推广成本太高，已经达到100元 ／个普通用户。
3、开发成本高，需要多个前后端程序员以及设计师，需要适应不同的手机设备。
4、不符合精益创业的原则。（三个主要工具是：“最小可用品”、“客户反馈”、“快速迭代”。）小成本快速小范围发布，尽快得到市场反馈，立即更改调整。
如何正确理解小程序？
1、它是微信的重点产品。
2、没有关注功能，对用户没有压力。
3、不是html，也不是混合代码，是自己独立的开发语言。
3、即用即走，随手可得。
4、一次开发，多端兼容。
5、微信会一如既往的限制营销。
6、离线能力。
微信是一个新生态？
1、微信是一个大平台。有统一入口，有独立语言，有统一规范，平台与开发者共赢。
2、需要新的应用场景，甚至新的国际用户。
3、原来在别的生态存在的应用，会以新的形态出现。
4、微信开发将来会是一个独立的岗位。
机会在哪里？
1、从别的生态中照搬过来。
2、补微信功能的缺口。
3、面对企业用户的创业。
4、新形势的营销。

听了知乎live关于编程入门学习的分享。

初学编程

编程

  • 对于计算机来说，有些事情解决起来非常简单困难，而另外一些则正相反，学会标志就是能够判断出一个问题，能否用编程来解决，并且对于自己要做的事情有一个很详细，但是大方向没有错的概念。

学编程概念

  • 那么具体到个人而言，想清楚自己所处领域里面用这些典型的问题可以通过计算机来解决，然后了解清楚哪些工具可以为你所用，想清楚工具是怎样的运作原理，其实就是学编程。学编程分为弄明白遇到一个问题怎样才能转化成写程序的思路，以及学会怎样才能又快又好地写出正确的程序来。这两个事情并没有固定的先后次序，一个人学编程的过程之中，这两个事情始终交织在一起，一方面弄清楚要解决的问题，会帮助理解为什么工具是那样的设计的。另一方面随着越来越熟悉工具，分析问题也会越来越顺利，这样一个工具和思路互相促进的过程，这个过程叫做建立编程思维。

判断自己文章好坏的标准

发表于 2016-11-01   |   分类于 笔记   |  

最佳问句问自己，我有没有让读者觉得……？

三条基础常识建议

多写

  • 多写才能知道自己的语病，自己的逻辑混乱，自己的平淡无奇，自己的不断进步。

认真思考所选品类的受众思考

  • 选定一个方向，高尔夫知识分享方向。受众者知识水平跨度非常大，这个领域的消费者和从业者完全就是一线城市和四线乡村的人的财富，等级，知识文化，见识水平等差距。要想明白真正对读者有用的是什么，不要被一些人误导，某些人的感觉没用，和感觉某些垃圾才有用，不应该动摇作者。对高尔夫爱好者真正有用的东西就是高尔夫方方面面的知识，那些知识从高尔夫被发明以来就被不断总结，那些文化从1457年就开始被传播。它们的有效性远远超过时下的八卦新闻和赛况战报。却由于英文资料整理困难，没有新鲜噱头被人忽视。坚决不为了关注度和传播效果去写感觉有用，甚至实则有害的东西。
  • 再深究一步，到底写运动相关的内容，到底什么对于读者最重要？
  • 我认为有两点：1.如何参与这项运动？2.运动中与个人成长和生活想通的部分是什么？
  • 让读者能通过文字融入这项运动，又通过运动反思到自己的成长和生活，那一定是读者最热爱的文字，看后更加理解和热爱这项运动，更能享受和升华生活。所以文章要分为运动的具体知识部分和这部分对思维和生活的影响。
  • 说理性的文章需要认真思考三要素
    简单vs复杂
    已知vs未知
    次要vs重要

三门问题，又被称为蒙提霍尔问题（Monty Hall Problem），它源自一个美国的电视游戏节目：Let’s Make a Deal。

做自己的CEO笔记第四周

发表于 2016-10-24   |   分类于 笔记   |  

经典经济学有两个假设：
1.信息总体是对称的，你知道的我也知道。
2.人总体是理性的，都会做对自己做有利的事。
但是行为经济学告诉我们，人在很多的情况下，并不那么理性。

结果偏见

  • 我们看到一个人取得了成功，就立刻认为他过去所有的行为都那么的有道理。可成功者认为的那个经验，可能是他获得更大成功的绊脚石。
    不能不管黑猫白猫，抓住老鼠的就是好猫，要分得清哪些是实力，哪些是运气。
  • 如何避免结果偏见？
    1.在归纳法上加上演绎法。从结果推到原因之后，要再看原因是否能真的推出正确的结果。
    2.做前3问。
    这个结果真的有一个人为可控的因素存在么？
    这个人，真的知道那个人为可控的原因是什么么？
    他引以为豪的，是不是恰恰是他的瑕疵呢？
  • 结果偏见就是外人看成功者羡慕，看失败者惋惜。
  • 归纳法让我们像一个律师一样，看到了结果，到事件中寻求支撑成功的证据。演绎法让我们像一个科学家一样，有了证据，再去推结论。如果只有归纳没有演绎，对总结出结论就会像情人眼里出西施，怎么看都对。
  • 高概率的成功方法比某一次成功的结果更重要。

首先跟大家说清楚今天的主题：如何打造且不断升级我们的操作系统，我们说的是阅读操作系统。大家一定要清楚我们讨论的是知识性阅读，不是娱乐性阅读，读诗歌、小说、散文虽然同样需要我们今天提到的这些技能，但我们今天的讨论聚焦在如何通过阅读获得知识进而改变生活。其次，我用了操作系统这个词，其实就是一个类比：我们可以把我们自己想象成一台电脑，至于这台电脑是古董级还是超级电脑，是靠我们自己的，或者说从一开始就是由我们自己决定的。

李笑来老师知乎Live《我的读书经验》全文

发表于 2016-10-23   |   分类于 笔记   |  

1.我的读书路径与境界更新

读书这件事在最初的时候很难做到目标明确，而且，一上来就目标明确的话，很可能会造成另外一个严重后果，即：失去获得意外好运的机会。所以，在第一个阶段里，我觉得甚至直接就应该漫无目的的读书，能读多少就读多少，能读到什么就读什么。不用挑，因为挑书也是一种逐步积累的能力。回顾我的读书经历，大约在28岁之前，我读书是非常杂的，连那时候男生都不屑于读的言情小说我都可以读的津津有味，现在回头看，这种饥不择食有很大的好处，随便罗列也起码有这么3个：

1.阅读变成了吃饭一样的需求，不读脑子就饿……

2.经常有各种意外的好运，从莫名其妙的地方读到了原本可能不可能获得的知识。

3.一不小心就积累了足够的里程—— 你要知道，驾驶经验远比驾驶技巧重要得多……

我有一个当图书馆馆长的妈妈，所以从小就懂一些基本的选书原则，因为我早就公开写过了，所以在这里我就不重复了，大家只要关注我的微信公众号【学习学习再学习】，在后台回复“选书”两个字，就可以看到我那篇文章：《选书的基本原则》。

到了第二个阶段，你就不得不挑书了，因为挑书的能力是在大量的阅读过程当中必然逐步形成，我从来都不向别人索要书单，因为最初就觉得书单是没有什么用的，那肯定是别人的书单，那肯定不是我的书单。后来，就庆幸了，因为从来不向别人索要书单，于是，思考久了观察多了，就明白一个特别简单的道理：向别人索要书单是使自己的挑书能力逐步退化直至为零的最有效方法。
